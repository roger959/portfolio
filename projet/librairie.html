📚 Librairie API – Projet individuel

📜 Description du projet

Lors de ma première année, j'ai réalisé un projet individuel intitulé Librairie API, un site web de consultation de livres à partir d’une API. Ce site permettait de parcourir un catalogue riche avec des fonctionnalités comme la recherche, la consultation de détails et l’ajout de livres en favoris.

🔧 Technologies utilisées

  • PHP – pour la logique côté serveur
  • CSS – pour le design de l’interface
  • JavaScript – pour interagir avec l’API
  • XAMPP – pour héberger en local et gérer la base de données
  • GitHub – pour la gestion de versions

🔐 Connexion sécurisée, base de données et gestion utilisateur

L'un des aspects majeurs de ce projet a été la mise en place d'un espace de connexion sécurisé, permettant aux utilisateurs d'accéder à un compte personnel. L’interface de cette section a été pensée pour être simple d’utilisation, tout en respectant les règles de sécurité et de validation côté serveur. Les utilisateurs devaient saisir un nom d'utilisateur et un mot de passe, et des contrôles étaient en place pour éviter les erreurs de saisie ou les injections SQL.

Une fois connecté, l'utilisateur a également la possibilité de modifier ses informations personnelles, comme son nom d'utilisateur ou son mot de passe. Pour cela, une interface dédiée a été développée avec des champs préremplis et une vérification côté serveur.

Cette fonctionnalité assure que les données modifiées sont validées et sécurisées avant d’être enregistrées dans la base de données. Par exemple, le mot de passe est automatiquement haché avec un algorithme de cryptage (comme password_hash de PHP) avant stockage.

Cela permet à chaque utilisateur de gérer son compte de manière autonome et en toute sécurité.

Pour rendre cette fonctionnalité possible, j’ai utilisé XAMPP comme environnement de développement local. Grâce à ce serveur Apache/PHP intégré, j’ai pu simuler un hébergement réel sur mon poste de travail et gérer efficacement les interactions entre le site et la base de données SQL.

J’ai conçu une base de données SQL structurée pour stocker les informations des utilisateurs (identifiants, mots de passe hachés, données de session) ainsi que les livres favoris. Le modèle relationnel comprenait des tables telles que :

  • utilisateurs : pour stocker les identifiants, mots de passe et rôles
  • livres : pour les métadonnées récupérées via l’API (titre, auteur, image, description…)
  • favoris : pour gérer la relation entre utilisateurs et livres enregistrés

Cette partie du projet m’a permis de consolider mes compétences en sécurité web, gestion des bases de données relationnelles et développement backend avec PHP.

⭐ Système de favoris

Une fonctionnalité d’ajout aux favoris a été intégrée : chaque utilisateur connecté pouvait ajouter des livres à sa liste personnelle pour les retrouver plus tard.

🧠 Métadonnées et API

Grâce à l’utilisation de l’API, chaque livre affichait des métadonnées telles que le titre, l’auteur, la description, l’éditeur et l’image de couverture. Cela permettait une navigation riche et interactive.

🛠️ Compétences développées

  • Créer un espace membre avec connexion/déconnexion
  • Manipuler une base de données avec XAMPP
  • Faire des requêtes vers une API externe
  • Ajouter des fonctionnalités personnalisées comme les favoris
  • Travailler seul en autonomie sur toutes les parties du projet

🎓 Conclusion

Ce projet m’a permis de combiner mes connaissances en développement web, en gestion d’API et en base de données. Il m’a aussi permis de structurer un site complet avec une logique utilisateur avancée tout en travaillant en autonomie.
